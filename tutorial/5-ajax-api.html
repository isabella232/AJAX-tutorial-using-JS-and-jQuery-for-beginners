AJAX API

An API, stands for Application Programming Interface and basically its something that allows one piece of software to another.

There are many kinds of APIs, but the one which you will deal most often is REST API where REST stands for REpresentational State Transfer. We don't have to bother with this for now.

I like to think of API using ships, docks and harbour. In order to use api we need to have as server (a harbour) which we can call.

Each call (ship) cary some stuff (data) to a harbour and get some data back. Sometimes it can carry very little data on the way to harbour (request) and bring lot of data back (response), and sometimes it can be oposite.

In order to make a proper call we have to specify a proper endpoint (dock in the harbour), otherwise our ship won't receive any data back.

Let's check the example:

Weather forecast

Let's imagine that we want to show our users a weather forecast. We will make use of free API available at openweathermap.org

Note! In order to use free API you have to register first, and get your API key from here. Free account allows us to call up 60x times per minute.

Now when we have API Key ready we can use one of the endpoint available to use

I am gonna use Current Weather data, according to docs I have to use the following endpoint: 


    api.openweathermap.org/data/2.5/weather?q={city name}&appid={your api key}

As you can see it contains some parameters, which we have to pass. Let's fill it with data: 


    api.openweathermap.org/data/2.5/weather?q=London&appid=ABC123

NOTE! You have to replace ABC123 key with yours, otherwise it won't work.

Now when we have endpoint ready, let's make a call: 


    //OPEN - call type, url/file , sync/async 
    xhr.open('GET','http://api.openweathermap.org/data/2.5/weather?q=London&appid=ABC123',true);

Voila! You made your first call to API. As you can see we are getting temperature in imperial units, what if we want to change it to metric?

We can simply add new parameter to our call: &units=metric


    api.openweathermap.org/data/2.5/weather?q=London&appid=ABC123&units=metric

You probably already noticed that while using GET request we are specifying parameters after URL using question mark sign (?), in a key=value syntax, and use & sign to split parameters.
